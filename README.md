# GitHub Pages Portfolio Starter

This is a simple static portfolio site designed for government or public-sector job applications.
It avoids unnecessary dependencies so it can be uploaded directly to a GitHub repository and published with GitHub Pages.

## Included files

- `index.html` — homepage with summary, featured projects, skills, and contact block
- `projects/` — three ready-to-edit case-study pages
- `assets/css/styles.css` — site styling
- `assets/js/main.js` — small script for the footer year

## How to publish on GitHub Pages

1. Create a new GitHub repository.
2. Upload all files in this folder.
3. In the repository, open **Settings** → **Pages**.
4. Under **Build and deployment**, choose **Deploy from a branch**.
5. Select the `main` branch and the `/ (root)` folder.
6. Save.
7. GitHub will publish the site at a URL like:
   `https://yourusername.github.io/repository-name/`

## What to customize first

1. Replace the homepage summary on `index.html`.
2. Update contact details on the homepage.
3. Replace the three sample projects with your real projects.
4. Add screenshots to `assets/img/` and reference them in the case-study pages.
5. Add a live GitHub Pages URL and repository link to each web project case study.
6. Update the PDF portfolio link or remove it if not needed.

## Tips for government applications

- Keep project titles plain and direct.
- Mirror the wording used in the job announcement where accurate.
- Show your role clearly.
- Use concise outcome statements.
- Make sure every link works without login.

## Optional additions

- Add a downloadable PDF portfolio.
- Add a resume PDF.
- Add more project pages by copying an existing page in `projects/`.

## Included usability fixes

- Anchor links include a built-in scroll offset so the sticky header does not cover section headings.
- Web project case studies include a dedicated Project Links section for a live GitHub Pages URL and repository URL.
